# Lessons

- Preserve exact requested and actual model IDs. “Frontier” is not a model receipt.
- A two-reviewer quorum should not hide a completed `NEEDS_WORK`; amend and re-gate the exact new artifact.
- Keep harness reward, fusion-gate verdict, and current-contract validation in separate columns.
- Pin plugin, runner, validator, dataset, image digest, host version, and timeout contract. Any one can drift.
- Publish failed and preflight-blocked attempts. They reveal integration defects that a successful provider smoke cannot.
- Default billable jigs to refusal and require an explicit execution flag.
- Curate public evidence from an allowlist. Raw sessions and environment-bearing logs are not necessary to prove call/model/cost facts.
- A single task is a smoke test, not a leaderboard.
